# 🔄 文件更新檢查

請幫我檢查最近的變更，並提醒我可能需要更新的文件。

## 檢查流程

### 1. 分析最近的 Git 變更
```bash
# 檢查最近的提交
git log --oneline -10

# 檢查未提交的變更
git status
git diff
```

### 2. 識別影響範圍

請根據變更的檔案，告訴我：
- 變更了哪些功能模組？
- 是否涉及資料結構？
- 是否涉及 API？
- 是否涉及 Pipeline？

### 3. 列出需要更新的文件

**必須更新（🔴）**：
- [ ] 列出所有必須更新的文件
- [ ] 說明每個文件需要更新的原因
- [ ] 建議更新的內容

**建議更新（🟡）**：
- [ ] 列出建議更新的文件
- [ ] 說明更新的好處

### 4. 檢查文件同步狀態

請檢查：
- [ ] `docs/WORK_LOG.md` 的「近期更新」是否同步？
- [ ] 相關 spec 文件是否反映最新變更？
- [ ] 工作日誌是否需要更新？

### 5. 版本管理

提醒我：
- 哪些文件的版本號需要更新？
- 更新日期是否需要修改？

---

**請現在告訴我：**
1. 最近有哪些重要變更？
2. 哪些文件需要更新？
3. 優先順序如何？
