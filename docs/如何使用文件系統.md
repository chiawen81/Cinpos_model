## 🔧 維護工作清單

### 🔴 開發前須更新

| 變更內容 | 需要更新的文件 |
|---------|--------------|
| 新增/刪除欄位 | data_資料欄位定義.md + ml_特徵配置.yaml |
| 新增/修改 API | spec_web_api.md |
| 變更核心功能 | 對應的 spec_*.md |

### 🟡 開發後須更新

| 情況 | 需要更新的文件 |
|-----|--------------|
| 完成重要功能 | WORK_LOG.md（近期更新）<br> 在 docs/work_log/ 創建新的工作日誌 |
| 做出重要決策 | WORK_LOG.md（重要決策記錄） <br> ps. 通常是待處理 |


### 🟢 視情況更新
| 完成優化實驗 | WORK_LOG_票房預測模型優化流程.md |
| 發現新優化方向 | ml_模型優化路線圖.md |

### 📅 定期維護

#### 每週檢查
```
□ 檢查 WORK_LOG.md 是否與實際進度同步
□ 更新待辦事項狀態
□ 記錄本週重要進展
```

#### 每月檢查
```
□ 檢查所有文件連結是否有效
□ 檢查文件版本號是否需要更新
□ 整理並歸檔已完成的工作日誌
□ 檢查文件之間的一致性
□ 更新所有文件的版本號
```
---

### 📅 使用流程
1. 每次啟用claude時：查看 **WORK_LOG.md** 目前進度、待辦事項

2. 每次執行一個任務的前：
執行 `/dev-start`，claude會提醒查閱相關文件、確認規範

3. 開發期間：
任何時候都可以執行 `/docs-update` ，claude 會分析最近變更，提醒需要更新的文件

4. 一個任務告一段落時：
執行 `/dev-done`，claude會檢查需要更新的文件、建議 commit message
ps. 照理說，若有重大功能開發會在資料夾 `docs/work_log/` 下，創建新的工作日誌，並同步更新 `docs/WORK_LOG.md` 的「近期更新」
