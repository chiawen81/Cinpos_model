## 🎓 使用建議

### 新手（第一週）
```
1. 先讀 README.md 和 CLAUDE.md
2. 瀏覽所有 spec_*.md（不用全部記住）
3. 開發時查閱對應文件
```

### 熟練後（日常）
```
開發前 → 查閱文件確認規範
開發中 → 遇到問題查文件
開發後 → 更新變更的文件
```

### 協作時
```
溝通時 → 引用文件章節
決策時 → 記錄在 WORK_LOG.md
變更時 → 同步更新文件
```

## ⚠️ 注意事項

### ✅ 好習慣
- 程式碼和文件一起提交
- 發現錯誤立即修正
- 重要決策記錄在工作日誌

### ❌ 壞習慣
- 只提交程式碼不更新文件
- 憑記憶開發不查文件
- 發現問題不記錄
